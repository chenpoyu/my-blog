「Poyu，我今年的績效...是不是不太好？」

小王坐在會議室裡，看起來很緊張。

這是我第一次主導年度績效面談，我也很緊張。

**績效面談，可能是管理者最難的任務之一。**

做得好，可以激勵成長；做得不好，可能傷害關係、打擊士氣，甚至讓人離職。

## 第一次績效面談的災難

去年我還是 Tech Lead 時，參與過主管的績效面談（以被評者的身份）。

那次經驗很糟：

**主管：** 「你今年表現還可以，但有些地方需要改進。」

**我：** 「哪些地方？」

**主管：** 「就...溝通可以更好，程式碼品質也可以再提升。」

**我：** 「具體是指什麼？」

**主管：** 「就...整體來說吧，沒有特定的例子，但你知道的...」

**我離開會議室時，感覺很挫折：**
- 不知道自己哪裡做錯
- 不知道如何改進
- 覺得評價很主觀

**我決定：如果我當主管，絕對不要這樣。**

## 準備工作

今年輪到我主導績效面談，我花了很多時間準備。

### 1. 收集數據

我不想只憑「感覺」評分，所以我整理了一整年的數據：

**任務完成度：**
- 分配了多少任務
- 完成了多少
- 有多少準時、延期、未完成

**程式品質：**
- Code Review 的評價（我請 Tech Lead 幫忙整理）
- 產生的 bug 數量
- 被 Review 時提出的問題

**團隊協作：**
- 參與了哪些會議和討論
- 幫助了哪些人（從 Slack 記錄整理）
- 360度回饋（我請團隊成員互評）

**學習成長：**
- 學習了什麼新技術
- 參加了哪些分享會
- 主導了什麼專案

**這些數據，讓我的評估有憑有據。**

### 2. 設計面談結構

我設計了一個四階段的面談結構：

**第一階段：自我評估（15分鐘）**
讓成員先說自己今年做得如何

**第二階段：主管回饋（20分鐘）**
我分享我的觀察和數據

**第三階段：落差討論（15分鐘）**
如果有不同看法，討論為什麼

**第四階段：明年目標（10分鐘）**
設定明年的改善方向

**總時間：約1小時**

### 3. 準備正面和改善點

我給每個人準備了：
- 3個做得好的地方（具體例子）
- 2個需要改善的地方（具體例子）

**原則：先說好的，再說改善。**

而且改善點不要太多（不超過3個），否則會讓人overwhelmed。

## 小王的績效面談

小王是 Mid-level 工程師，今年表現中等。

有進步，但也有問題。

### 第一階段：自我評估

**我：** 「小王，你覺得你今年表現如何？」

**小王：** 「我覺得...還可以吧。完成了大部分任務，也學了一些新東西。」

**我：** 「如果給自己打分數，1-10分，你會打幾分？」

**小王：** 「7分吧。」

**我：** 「為什麼是7分？」

**小王：** 「因為有些任務沒做好，而且感覺沒有太大突破。」

**我：** 「你覺得哪裡可以做得更好？」

**小王：** 「可能...程式碼品質？還有溝通？」

**觀察：他對自己的評價還算客觀。**

### 第二階段：主管回饋

**我：** 「我先說你做得好的地方。」

**1. 任務完成度很高**

「今年你被分配了 25 個任務，完成了 23 個，完成率 92%。而且有 18 個是準時完成，這很不錯。」

（拿出數據表格給他看）

**小王：** 「喔，我沒想到有這麼高。」

**2. 主動幫助新人**

「今年你帶了小李，他進步很快。我問過他，他說你很有耐心，會手把手教他。這對團隊幫助很大。」

**小王：** 「應該的啦，我也是這樣被帶過來的。」

**3. 學習意願高**

「你今年學了 Docker 和 Kubernetes，還主動分享給團隊。這種學習態度很棒。」

**小王：** 「謝謝Poyu。」

**（先說好的，讓他有信心，再說改善點）**

**我：** 「但也有兩個地方需要改善。」

**1. Code Review 的細心度**

「今年你有 5 個PR在上線後發現bug，其中 3 個是可以在 Code Review 階段抓到的。」

（拿出具體例子）

「比如這個，你沒有檢查 null 值，導致 NullPointerException。」

**小王：** 「對...這個我記得，當時太趕了。」

**我：** 「理解，但這就是需要改善的地方。即使趕，也要確保品質。」

**2. 溝通的主動性**

「有幾次你遇到問題，但沒有及早提出，導致任務延期。」

（拿出具體例子）

「比如9月那個需求，你卡住了兩天，但沒有跟我說，最後延期了。」

**小王：** 「我以為我可以自己解決...」

**我：** 「我知道，但卡住超過半天，就應該尋求協助。這不是能力問題，是溝通問題。」

### 第三階段：落差討論

**我：** 「整體來說，我給你 7.5 分。你覺得合理嗎？」

**小王：** 「嗯，合理。我本來以為會更低，因為有些任務做得不好。」

**我：** 「你做得好的地方比你想的多，不要太低估自己。但也要注意改善的點。」

### 第四階段：明年目標

**我：** 「明年，我希望你在這兩個方面改善：」

**1. 提升 Code Review 的細心度**
- 目標：減少 PR 上線後的 bug 數量（從5個降到2個以下）
- 方法：使用 checklist、花更多時間檢查

**2. 提升溝通的主動性**
- 目標：遇到問題及早反映（卡住超過半天就要說）
- 方法：定期更新進度、主動尋求協助

**我：** 「這兩個目標可以達成嗎？」

**小王：** 「可以，我會努力。」

**我：** 「好，我們下次季度檢視時，再看看進度。」

**面談結束，小王看起來鬆了一口氣，也知道明年要改善什麼。**

## 小陳的績效面談（困難案例）

小陳是資深工程師，今年表現不好。

任務完成度低、態度消極、影響團隊士氣。

**這是最難的面談。**

### 第一階段：自我評估

**我：** 「小陳，你覺得你今年表現如何？」

**小陳：** 「還可以吧...完成了一些任務。」

**我：** 「如果打分數，你會打幾分？」

**小陳：** 「6分吧。」

**我：** 「為什麼只有6分？」

**小陳：** 「因為...今年沒有太多挑戰性的任務，做得不是很起勁。」

**（他在為自己找藉口）**

### 第二階段：主管回饋

**我：** 「我先說你做得好的地方。」

**1. 技術能力強**

「你的技術沒問題，遇到難題通常都能解決。」

**小陳：** 「嗯。」

**（我只能找到一個優點...）**

**我：** 「但有幾個地方需要認真改善。」

**1. 任務完成度低**

「今年你被分配了20個任務，只完成了12個，完成率60%。而且有6個延期。」

**小陳：** 「有些任務不合理，需求一直改。」

**我：** 「我理解需求會改，但其他人也是同樣的情況，完成率都在80%以上。」

**（拿出數據對比）**

**小陳：** 「...」

**2. 態度消極**

「有幾次會議，你表現出不耐煩，甚至質疑專案的意義。這影響了團隊士氣。」

**小陳：** 「我只是覺得有些專案沒意義...」

**我：** 「你可以有意見，但表達方式要建設性。抱怨不會改變什麼，只會讓大家都不想做。」

**3. 不願意配合**

「幾次我請你協助新人或參與討論，你都推辭或敷衍。」

**小陳：** 「我有自己的任務要做...」

**我：** 「資深工程師的責任不只是完成任務，還包括幫助團隊成長。」

### 第三階段：落差討論（關鍵時刻）

**我：** 「小陳，我想聽聽你的想法。為什麼今年表現不如以前？」

**小陳：** 「說實話，我覺得這裡沒有挑戰，每天做重複的事，沒意思。」

**我：** 「你有跟我說過嗎？」

**小陳：** 「沒有...我以為你知道。」

**我：** 「我不知道，除非你告訴我。」

**（沉默）**

**我：** 「如果你覺得沒有挑戰，我們可以討論。但消極應對、影響團隊，這不是解決方式。」

**小陳：** 「那現在有什麼挑戰可以給我？」

**我：** 「我可以創造機會，但你要先改善態度。我不會把重要專案交給一個消極的人。」

**（直接但誠實）**

**小陳：** 「那我要怎麼改？」

**我：** 「首先，提升任務完成度。其次，改善態度和配合度。如果你做到了，我會給你更有挑戰性的任務。」

### 第四階段：明年目標（附帶警告）

**我：** 「小陳，我給你明確的目標：」

**1. 任務完成度提升到 80% 以上**
**2. 停止消極言論和行為**
**3. 主動協助團隊（每月至少2次）**

**我：** 「如果明年Q1你沒有改善，我會建議你換部門或換工作。我說的是真的。」

**小陳：** 「我知道了。」

**（他臉色凝重，但我必須說清楚）**

## 小李的績效面談（正面案例）

小李是 Junior，今年進步很多。

**這是最愉快的面談。**

**我：** 「小李，你今年進步很大，從什麼都不會，到現在能獨當一面。」

**小李：** 「謝謝Poyu，謝謝小王的幫忙。」

**我：** 「你的學習態度很好，而且不怕問問題。這是正確的。」

**小李：** 「那我明年可以升 Mid 嗎？」

**我：** 「可以，如果你能做到這幾點：」
- 獨立完成中等難度的任務
- 能指導其他新人
- 能參與技術決策

**小李：** 「我會努力！」

**（這種面談很簡單，就是肯定+鼓勵+設定目標）**

## 我學到的事

做了10幾場績效面談，我學到：

**1. 數據很重要**

有數據支撐，評價才有說服力。

「我覺得你不夠努力」沒有說服力。  
「你今年完成率只有 60%，低於團隊平均 85%」有說服力。

**2. 先說好的，再說改善**

如果一開始就批評，對方會防禦性地抗拒。

先肯定，讓他放鬆，再提改善，他會更願意接受。

**3. 具體的例子**

「你溝通不好」太模糊。  
「9月那個需求，你卡住兩天沒說，導致延期」很具體。

**4. 設定可達成的目標**

不要一次給太多改善目標，會讓人overwhelmed。

2-3個就好，而且要具體、可衡量。

**5. 誠實但不殘忍**

對於表現不好的人，要誠實地說出問題。

但也要給機會改善，而不是直接否定。

**6. 追蹤和支持**

績效面談不是年度儀式，而是持續的過程。

每季檢視進度，提供支持，確保改善。

## 反思

績效面談，其實是「關係經營」的一部分。

做得好，可以建立信任、激勵成長。  
做得不好，可能破壞關係、打擊士氣。

**最重要的是：真誠和公平。**

真誠：不虛偽，不模糊，誠實地反映事實。  
公平：用一致的標準，不偏袒，不情緒化。

如果團隊成員覺得「這個評價是公平的」、「主管真的在乎我的成長」，他們就會接受，甚至感激。

**這才是績效面談的真正意義。**
