今天面試了一位技術能力很不錯的候選人，但整場面試都很卡，卡到我一度懷疑是不是我的問題。

## 尷尬的面試過程

他履歷上寫的經歷很豐富，做過幾個大型專案，技術棧也很全面。我心想應該是個不錯的人選。

面試開始，我先請他自我介紹。他只簡短說了名字和工作年資，然後就停了。我等了幾秒，他沒有繼續。

我只好主動問：「可以聊聊你之前做過的專案嗎？」

「嗯...做過電商系統。」  
「那個專案你負責什麼部分？」  
「後端。」  
「後端的哪些模組？」  
「訂單、支付...那些。」

每個回答都超級簡短，我得一直追問才能多了解一點。這種感覺就像在拷問犯人一樣，很累。

## 技術能力確實不錯

為了更客觀評估，我給了他一個實作題：「寫一個函式，判斷一個字串是否為迴文。」

他拿起白板筆，刷刷刷就寫完了：

```java
public boolean isPalindrome(String str) {
    int left = 0;
    int right = str.length() - 1;
    while (left < right) {
        if (str.charAt(left) != str.charAt(right)) {
            return false;
        }
        left++;
        right--; 
    }
    return true;
}
```

程式碼寫得很乾淨，邏輯也清晰，時間複雜度是 O(n)，沒什麼問題。

我想了解他的思路，問：「為什麼選擇用雙指標的方式？」

他想了一下，說：「就...這樣比較好。」

「比較好是指什麼？效能？還是可讀性？」

「嗯...都有吧。」

我又問：「如果這個字串很長，有沒有其他優化方式？」

「應該...可以吧。」

然後又停住了。

## 團隊協作的考量

技術測驗的部分確實沒問題，但溝通能力真的是個大問題。

我嘗試問一些情境題：「如果你和同事對某個技術選型有不同意見，你會怎麼處理？」

「看情況。」  
「什麼樣的情況？」  
「就...看哪個比較好。」  
「怎麼判斷哪個比較好？」  
「...測試看看？」

整場面試下來，我感到非常疲憊。不是因為他能力不好，而是溝通真的太困難了。

## 為什麼沒有錄取

面試結束後，我和 HR 討論了這個案例。

我們團隊現在有很多協作需求：
- 需要跟客戶溝通需求
- 需要跨組討論技術方案
- 需要在週會上分享進度
- 需要帶新人和做 Code Review

如果一個人無法清楚表達想法，這些工作都會很困難。再強的技術，如果無法分享、無法討論，對團隊的價值就會大打折扣。

尤其我們有 60% 的人是派遣去客戶端，如果連基本的溝通都有問題，客戶那邊一定會有意見。

所以最後決定不錄取。

## 反思：溝通能力可以培養嗎？

但這個決定也讓我反思了很久。

**溝通能力真的無法培養嗎？**

有些人可能只是比較內向或緊張，並不代表永遠都無法溝通。如果給他時間和環境，也許可以改善？

**我們是否過於重視溝通，而忽略了技術？**

在某些純技術的崗位，也許溝通能力沒那麼重要？但我們團隊目前確實沒有這種純研發的角色。

**是否該給一次機會，試用期觀察？**

但試用期如果還是不行，對雙方都是浪費時間。

這個問題沒有標準答案。不同團隊、不同情境，可能會有不同的選擇。

對我們現在的團隊來說，溝通能力是必要的。但我也會記住這個案例，提醒自己：面試評估要全面，技術只是其中一部分。
