新的一年準備了一個小小的計劃養成，想利用工作中遇到的事情進行記錄，讓我將學習與收穫到的事情能有所保留，因此開始了每個月的一篇文章紀錄，不管長短內容，就是一種練習！

目前的想法是，不刻意去寫長篇大論的技術文件，而是把每天在 Java 專案裡踩到的坑、看到的爛 Code，或是前輩點出我的架構問題，用自己的話整理出來。

這是一份給日後自己的回顧，也希望透過這種強迫輸出的方式，讓自己在 OOAD 的思維上能更有邏輯一點。

**目前的目標：**
1. 針對工作中的電商邏輯進行重構練習。
2. 保持每個月至少產出一篇，紀錄開發上的轉變。
3. 透過實作 Java 設計模式來解決真實問題。

先從最常見的 `if-else` 開始吧，希望一年後回來看這篇文章，我已經不再是那個只會寫麵條代碼的工程師了。
