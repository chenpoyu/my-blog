## 最近真的太忙了

Q4 的專案一個接一個，加上年底收尾的一堆行政雜事，整個人被壓到有點喘不過氣。

這種時候我的習慣不是去放空（放空完回來只會更焦慮），而是做一些**跟工作無關但有成就感的小東西**。自己用的工具、讓生活方便一點的自動化，什麼都好。用一個不同的腦在運轉，反而是某種休息。

所以就動了一個一直想做的東西：讓 AI 每天自動教我英文。

## 想法很簡單

我英文不算差，但一直停在那個「夠用但不夠好」的尷尬地帶。尤其是比較正式的商務場合跟學術寫作，用字遣詞常常覺得不夠到位。之前有想過要不要買線上課程或找家教，但以我現在的工作節奏，根本不可能固定時間上課。

那不如做一個機器人，每天自動推一課給我。通勤的時候看、午休的時候看、睡前花五分鐘看，零碎時間就能用。

## 架構：n8n + Ollama + Slack

用 n8n 做排程和流程編排，Ollama 跑本地 LLM 來生成教學內容，最後推到 Slack。

為什麼用本地 LLM？因為這是每天都要跑的東西，用 OpenAI API 的話日積月累也是一筆費用。Ollama 跑在自己機器上，不花錢，延遲也能接受。

流程很直覺：

1. n8n 每天早上固定時間觸發
2. 從 CSV 讀取當天的主題（我事先規劃了 365 天的主題）
3. 把主題塞進 prompt，丟給 Ollama 生成內容
4. 格式化之後推到 Slack 頻道

## Prompt 是花最多心思的地方

生成品質好不好，九成取決於 prompt 怎麼寫。我設定的角色是一位「托福學術英文 + 高階商務英文」的雙語導師，每天根據主題產出一份完整的教學內容。

結構大概是這樣：

**一、核心關鍵詞**：挑 3 個托福難度的單字，必須跟當天主題高度相關，每個附商務例句。

**二、商務模擬情境**：一段 4-6 句的英文對話，自然融入那 3 個單字。重點是做成英中對照，中文除了翻譯，還會標註那句話在職場裡的潛台詞或語氣。這個設計是因為很多英文對話，直翻中文看不出來它其實很委婉或很強勢。

**三、深度解析**：從對話裡拉出核心句型，講文法結構，也講這個句型在托福寫作或正式信件裡怎麼用。

**四、每日挑戰**：一題中翻英，搭配標準答案跟解析。

在 prompt 裡我也下了幾個硬性限制：對話裡的關鍵單字要加粗、所有解釋用繁體中文、單字要是托福範圍不能太基礎。這些約束不寫清楚的話，LLM 很容易開始放水。

## 365 天的主題規劃

這個是前期準備最花時間的部分。我把主題放在一份 CSV 裡，n8n 每天讀取對應的那一行。

主題的規劃大致按月份分：

- 一月：職場溝通基礎（會議、報告、信件）
- 二月：談判與說服
- 三月：跨部門協作
- 四月：專案管理
- ⋯⋯依此類推

每個月有一個大方向，每天是一個小切面。這樣學起來有脈絡，不會今天聊財報明天突然跳到面試。

做這份清單的時候，真的覺得自己像在排課綱。不過排完之後就不用再管了，n8n 會自動按表操課。

## 實際跑起來的效果

跑了幾天之後，品質比我預期的好。Ollama 用的模型雖然不是最頂級的，但在這種結構化輸出的任務上表現得還不錯。偶爾會有單字選得太簡單的情況，但大部分時候都還行。

每天早上開 Slack 就有一課在等我，那個感覺蠻好的。不用自己去找教材、不用打開 App、不用付費訂閱，就是一個安安靜靜待在那邊的英文老師。

後來組裡有幾個人看到我的 Slack 頻道，也說想加入。結果那個頻道變成部門裡一個小小的英文讀書會，每天有人會回一句「今天這個單字我還真的不會」之類的。算是意外的收穫。
