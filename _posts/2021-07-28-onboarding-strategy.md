**以下資訊，若涉及個人資訊，會以模糊方式調整，避免被帶入特定人士縮影**

團隊從年初的 12 人成長到現在快 30 人，平均每個月都有 2-3 位新人加入。如何讓新人快速上手，成了我最頭痛的問題之一。

一開始我很樂觀，覺得新人來了給他們看文件、跟著資深的學就好。但實際運作後發現，根本不是這麼回事。

## 最初的混亂

記得第一批新人加入時的情景。

第一天，新人 A 來報到，我把他交給資深工程師小張：「你帶一下他，教他怎麼建環境。」小張說好。

第二天，新人 B 也來了，小張還在教 A，我只好找另一位資深工程師小李幫忙。

第三天，新人 C 又來了，這時候小張和小李都忙著自己的專案，我只好自己教。

到了第四天，我發現：
- 新人 A 還在裝開發環境，遇到一堆奇怪的錯誤
- 新人 B 環境是裝好了，但不知道要做什麼
- 新人 C 拿到任務了，但看不懂代碼結構

更慘的是，三個新人問的問題都差不多，但我們三個人各自教一遍，浪費了大量時間。

這時候我才意識到：**新人訓練需要系統化。**

## 建立 Onboarding 流程

我花了一個週末，整理出一套四週的 Onboarding 流程：

### 第一週：環境與文件

**目標：讓新人能夠在本機跑起專案**

- Day 1-2：安裝開發工具（JDK, Maven, IDE, Git）
- Day 3-4：Clone 專案，跑起來
- Day 5：閱讀專案文件，了解整體架構

**我做的改善：**
- 寫了詳細的環境建置文件，包含常見問題的解法
- 準備一個 Docker 版本，讓新人可以快速跑起來
- 錄製環境建置的影片，新人可以邊看邊做

這樣一來，新人遇到問題可以先查文件和影片，不用一直問。

### 第二週：小任務練手

**目標：讓新人熟悉開發流程**

給新人一些簡單的任務，像是：
- 修改一個文字內容
- 新增一個簡單的 API endpoint
- 修復一個容易的 bug

**重點是：**
- 任務要簡單，但要走完整個流程（開分支、寫代碼、提 PR、Code Review、合併）
- 讓新人熟悉我們的開發規範和工具

這週結束時，新人至少要能獨立提一個 PR。

### 第三週：參與正式專案

**目標：開始做有價值的工作**

讓新人加入某個專案，但還是以簡單的任務為主：
- 跟著資深工程師做一個功能的某個部分
- 參加每日站會，了解專案進度
- 嘗試自己看懂需求和設計文件

這週新人會開始覺得自己是團隊的一份子，不只是在學習。

### 第四週：獨立負責模組

**目標：能夠獨立完成任務**

給新人一個小模組，讓他獨立負責：
- 從需求理解到設計實作
- 有問題可以問，但要自己先想
- 完成後要自己測試

這週結束時，新人應該可以獨當一面（至少在簡單任務上）。

## Mentor 制度

光有流程還不夠，每個新人需要有人關照。

我建立了 Mentor 制度：每個新人配一位資深工程師當 Mentor，負責：

**Mentor 的職責：**
1. 回答新人的技術問題
2. Review 新人的代碼
3. 每週至少一次 1-on-1 聊聊狀況
4. 協助新人融入團隊

**但這也帶來問題：**

當新人越來越多，Mentor 也會忙不過來。一個 Mentor 同時帶兩三個新人，自己的工作就會受影響。

有個資深工程師跟我抱怨：「我現在一半時間在教新人，自己的專案進度都落後了。」

## 再次改善：文件化與標準化

我意識到，不能讓 Mentor 事事都教，要讓新人能自主學習。

**我們開始建立知識庫：**

1. **技術文件**
   - 專案架構說明
   - 常用模組的使用方式
   - API 設計規範
   - 資料庫 Schema 說明

2. **開發規範**
   - Coding Style Guide
   - Git Workflow
   - Code Review 標準
   - 測試規範

3. **FAQ 文件**
   - 環境建置常見問題
   - 開發常見錯誤
   - 部署注意事項

4. **影片教學**
   - 環境建置
   - 專案導覽
   - 常用工具教學

有了這些文件，新人遇到問題可以先自己查，真的找不到答案再問 Mentor。

## 持續優化

Onboarding 流程不是一次就定型的，需要持續收集反饋和改善。

每個新人在第一個月結束時，我會請他們填一份問卷：
- 哪個部分最困難？
- 哪些文件最有幫助？
- 哪些文件不夠清楚？
- 還需要什麼資源？

根據這些反饋，我們不斷優化流程和文件。

## 現在的效果

經過三個月的優化，現在新人的 Onboarding 順暢多了：

**數據對比：**
- 之前：新人平均 6-8 週才能獨立工作
- 現在：新人平均 4 週就能獨立工作

- 之前：Mentor 每天花 2-3 小時教新人
- 現在：Mentor 每天花 0.5-1 小時

- 之前：新人離職率約 30%（前三個月）
- 現在：新人離職率降到 10%

**新人的反饋也變好了：**
- 「文件很完整，大部分問題都能自己解決」
- 「任務安排很清楚，知道每週要做什麼」
- 「不會覺得迷失，能感受到進步」

## 還在面對的挑戰

雖然改善了很多，但還是有一些挑戰：

1. **文件維護** - 專案一直在變，文件要跟著更新，但常常忘記
2. **人力分配** - Mentor 還是會抱怨帶新人影響工作
3. **個別差異** - 有些新人學很快，有些需要更多時間，很難一套流程適合所有人

這些是下一階段要解決的問題。

## 反思

**新人 Onboarding 不只是讓他們學會工作，更是讓他們感受到被重視。**

一個好的 Onboarding 流程會讓新人覺得：
- 公司很專業，有系統
- 我不是被丟著自生自滅
- 有人在乎我的成長

這些感受會影響新人是否願意長期留在團隊。

投資在 Onboarding 上的時間，長期來看絕對值得。
